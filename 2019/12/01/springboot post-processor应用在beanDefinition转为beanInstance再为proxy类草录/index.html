еүҚжҸҗ

еҰӮжһңдҪ е·Із»ҸеҫҲдәҶи§Јspringboootдёӯbean post-processorзҡ„еҺҹзҗҶе’ҢдҪҝз”ЁпјҢжң¬ж–Үеҗ‘дҪ еұ•зӨәдәҶдёҖдёӘзІҫз®ҖзүҲе’ҢеҸҜи§ҶеҢ–еӣҫи°ұпјӣеҰӮжһңдҪ еҫҲжғідәҶи§Је®ғпјҢжң¬ж–ҮжҸҗдҫӣдҪ жўҜеӯҗеҝ«йҖҹзҡ„иө°е…Ҙbean post-processorеҺҹзҗҶе’Ңеә”з”ЁгҖӮд»ҘеҮҸе°‘дҪ еҺҹжң¬йңҖиҰҒиҠұиҙ№зҡ„ж—¶й—ҙ
жң¬ж–Үзҡ„з»“жһ„дёәеӣҫ+ж–Үжң¬з»“еҗҲзҡ„ж–№ејҸйҳҗиҝ°

зҹҘиҜҶзӮ№ж”¶иҺ·

  1. BeanPostProcessorдҪңз”Ё
  2. BeanPostProcessorдёӯе…¶жҜҸдёӘж–№жі•зҡ„еә”з”ЁеңәжҷҜ

  4. BeanPostProcessorзұ»еһӢеҜ№иұЎеңЁspring iocж—¶зҡ„еә”з”Ё

еӣҫдёӯеұ•зӨәдәҶspring(boot)дёӯpost-processorеә”з”ЁеңЁBeanDefinitionиҪ¬дёәBeanInstanceеҶҚдёәProxyд»ЈзҗҶзұ»ж•ҙдҪ“зҡ„жөҒзЁӢпјҢеҢ…жӢ¬е“ӘдёӘж–№жі•еә”з”ЁдәҶе…·дҪ“зҡ„е“ӘдёӘpost-processorпјҢд»ҘиҫҫеҲ°д»Җд№Ҳзӣ®зҡ„жҲ–дҪңз”ЁгҖӮ

зңӢе®Ңеӣҫдёӯзҡ„еӣҫдҫӢеҗҺеҶҚзңӢжӯЈеӣҫ

жӯЈж–Ү

зҺ°еңЁпјҢжҲ‘д»¬з»“еҗҲеӣҫзҡ„еҹәзЎҖдёҠпјҢйҳҗиҝ°eanPostProcessorзҡ„е®¶ж—Ҹе’ҢдҪҝз”Ё

BeanPostProcessorдҪңз”Ё

BeanPostProcessorжҳҜspringжЎҶжһ¶дёӯйқһеёёйҮҚиҰҒзҡ„зұ»пјҢжҜҸдёҖдёӘж”ҫеҲ°spring iocдёӯзҡ„beanзҡ„еҲӣе»әиҝҮзЁӢйғҪжңүд»–зҡ„иә«еҪұгҖӮдҝ—з§°еүҚзҪ®еӨ„зҗҶе’ҢеҗҺзҪ®еӨ„зҗҶпјҢж„ҸжҖқдёәеңЁbeanзҡ„еҲӣе»әеүҚеҗҺеҸҜд»ҘеўһеҠ иҮӘе®ҡд№үзҡ„еӨ„зҗҶйҖ»иҫ‘гҖӮеҰӮпјҢbeanеҲӣе»әеүҚз»ҷжҹҗдёӘзұ»еһӢзҡ„beanеўһеҠ дёҖдёӘеұһжҖ§еҖјпјҢеңЁbeanеҲӣе»әеҗҺз”ҹжҲҗиҝҷдёӘbeanзҡ„д»ЈзҗҶзұ»ж”ҫеҲ°spring iocдёӯзӯүзӯү

жіЁж„Ҹпјҡ BeanPostProcessorжҳҜеңЁspringе®№еҷЁеҠ иҪҪдәҶbeanзҡ„е®ҡд№үж–Үд»¶е№¶дё”е®һдҫӢеҢ–beanд№ӢеҗҺжү§иЎҢзҡ„гҖӮ BeanPostProcessorзҡ„жү§иЎҢйЎәеәҸжҳҜеңЁBeanFactoryPostProcessorд№ӢеҗҺгҖӮе…ідәҺBeanPostProcessor vs BeanFactoryPostProcessorпјҢеҸӮи§ҒBeanFactoryPostProcessoryдёҺBeanPostProcessorеҢәеҲ«

BeanPostProcessorдёӯе…¶жҜҸдёӘж–№жі•зҡ„еә”з”ЁеңәжҷҜ

зңӢBeanPostProcessorзұ»з»“жһ„

BeanPostProcessorжңүдёӘз»ҸеёёдҪҝз”Ёзҡ„еӯҗзұ»еһӢпјҡInstantiationAwareBeanPostProcessorпјҢзңӢе…¶з»“жһ„

InstantiationAwareBeanPostProcessorжҜ”BeanPostProcessorеӨҡзҡ„еҠҹиғҪжҳҜпјҡadds a before-instantiation callback, and a callback after instantiation but before explicit properties are set or autowiring occurs

е…ёеһӢзҡ„дҪҝз”ЁеңәжҷҜеҰӮ create proxies with special TargetSources (pooling targets, lazily initializing targets, etc), or to implement additional injection strategies such as field injection

InstantiationAwareBeanPostProcessorжҜ”BeanPostProcessorеӨҡдәҶдёүдёӘж–№жі•

д»ҺејҖзҜҮзҡ„еӣҫдёӯеҸҜзңӢеҮәпјҢbeanзҡ„еҲӣе»ә(beanDefinition->beanInstance->proxy)дёҺpost-processorзҙ§еҜҶзҡ„дәӨз»ҮеңЁдёҖиө·гҖӮеҰӮдҪ•жӯЈзЎ®зҡ„дҪҝз”Ёд»–д»¬жҲҗдёәйҡҫзӮ№е’Ңе…ій”®зӮ№

BeanPostProcessorе®¶ж—ҸжҲҗе‘ҳ

SpringдёӯеҶ…зҪ®дәҶеҫҲеӨҡзҡ„BeanPostProcessorе®һзҺ°зұ»пјҢеҲ—дёҫзҡ„йғҪжҳҜжҲ‘д»¬зҶҹжӮүзҡ„пјҢд»–д»¬з»ҷеҮәдәҶеҫҲеҘҪзҡ„дҪҝз”ЁдҫӢеӯҗпјҢдёәжҲ‘д»¬дҪҝз”ЁBeanPostProcessorеҺ»е®һзҺ°жҲ‘д»¬зҡ„иҮӘе·ұеңәжҷҜе’Ңеә”з”ЁжҸҗдҫӣдәҶеҸӮиҖғзҡ„е§ҝеҠҝ

пјӣеҗҢж—¶пјҢж–ҮдёӯиҜҰиҝ°дәҶBeanPostProcessorе’ҢInstantiationAwareBeanPostProcessorзҡ„BeforeInstantiationгҖҒAfterInstantiationгҖҒBeforeInitializationгҖҒAfterInitializationгҖҒpostProcessPropertiesж–№жі•зҡ„еә”з”ЁеңәжҷҜе’Ңеә”з”Ёж—¶жңәпјҢеёҢжңӣеҸҜд»Ҙеј•еҜјеҮәеҗҲйҖӮзҡ„дҪҝз”Ёе§ҝеҠҝ
